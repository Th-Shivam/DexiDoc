## Core Principles

1. **Local-Only by Default**

   * No cloud sync
   * No remote APIs
   * No telemetry unless explicitly enabled

2. **Explainable Actions**

   * Every operation is previewed
   * No silent or destructive changes

3. **Progressive Intelligence**

   * Starts simple and deterministic
   * Smarter features are optional and modular

4. **Minimal, Extensible Core**

   * Small, fast base system
   * Advanced capabilities added via plugins

## What DexiDoc Is (Phase 1 Scope)

In its first usable release, DexiDoc aims to:

* Index local documents
* Search them using natural, assistant-like commands
* Perform safe, basic file operations
* Work fully offline

Examples:

```text
find invoices from december
show files modified last week
open my resume
```

---

## What DexiDoc Is NOT (For Now)

To keep the project focused, DexiDoc intentionally does **not** include:

* Cloud storage or sync
* Collaboration or sharing features
* Always-on AI or remote LLMs
* Enterprise compliance tooling
* Automatic background uploads

These may be explored later â€” but are **explicitly out of scope** for early phases.
